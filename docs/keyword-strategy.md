# 키워드 전략 설계서

> **문서 목적**: 1페이지(40위) 진입 및 유지를 위한 키워드 발굴, 선정, 관리 전략을 정의한다.

---

## 1. 핵심 목표

```
"1페이지를 먹을 수 있는 키워드를 계속 발굴하고,
 그 중 인기도에 가장 기여하는 키워드들을 살려나간다"
```

### 1.1 1페이지의 의미

- 네이버 쇼핑 검색 결과 **상위 40위** (광고 제외)
- 40위 안에 들어야 실질적인 클릭/유입 발생
- 2페이지 이하는 거의 노출 효과 없음

### 1.2 단계별 목표

| 단계 | 현재 상태 | 목표 |
|------|----------|------|
| 극초반 | 대표키워드 500위+ | 롱테일 키워드 1페이지 진입 |
| 성장기 | 대표키워드 100~500위 | 중간 키워드 1페이지 진입 |
| 안정기 | 대표키워드 100위 이내 | 대형 키워드 1페이지 진입 |

---

## 2. 네이버 쇼핑 순위 공식

### 2.1 기본 공식

```
순위 점수 = 인기도 × 적합도(%) × 신뢰도(%)
```

| 요소 | 설명 | 우리가 할 수 있는 것 |
|------|------|---------------------|
| **인기도** | 검색 후 클릭, 검색 후 전환 (일주일 기준) | 판매가 일어나야 쌓임 (결과) |
| **적합도** | 검색 키워드와 상품 정보 일치도 | 상품명/태그 최적화 (조작 가능) |
| **신뢰도** | 스토어 평판, SEO 준수 | 시간이 걸림 (관리만 하면 OK) |

### 2.2 전략적 의미

1. **적합도**를 높여서 노출 확보
2. 노출로 **클릭/전환** 발생
3. 클릭/전환으로 **인기도** 축적
4. 인기도로 **더 큰 키워드** 도전 가능

```
[선순환 구조]

적합도 최적화 → 노출 → 클릭/전환 → 인기도 상승 → 더 큰 키워드 도전
      ↑                                              │
      └──────────────────────────────────────────────┘
```

## 3. 키워드 발굴 소스

### 3.1 세 가지 발굴 소스

| 소스 | 방법 | 데이터 |
|------|------|--------|
| **1. 상품명 분석** | 내 상품명 토큰화 | 현재 노출 가능 키워드 |
| **2. 검색광고 API** | `/keywordstool` 연관 키워드 | 검색량 + 경쟁강도 |
| **3. 경쟁사 분석** | 상위 40개 상품명 크롤링 | 실제 1페이지 키워드 |

### 3.2 소스별 상세

#### 3.2.1 상품명 분석 (내 상품)

```
입력: "볼펜 퀵드라이 젤펜 중성펜 선물용"

토큰화: ["볼펜", "퀵드라이", "젤펜", "중성펜", "선물용"]

조합 생성:
  - 2토큰: "볼펜 퀵드라이", "퀵드라이 젤펜", ...
  - 3토큰: "볼펜 퀵드라이 젤펜", ...
  - N토큰: ...
```

#### 3.2.2 검색광고 연관 키워드 API

```
GET /keywordstool?hintKeywords=볼펜

응답:
[
  { relKeyword: "볼펜", monthlySearchCount: 50000, compIdx: "HIGH" },
  { relKeyword: "젤펜", monthlySearchCount: 8000, compIdx: "MEDIUM" },
  { relKeyword: "퀵드라이 젤펜", monthlySearchCount: 500, compIdx: "LOW" },
  ...
]
```

**활용 데이터:**
- `monthlyPcQcCnt + monthlyMobileQcCnt`: 월간 검색량
- `compIdx`: 경쟁강도 (LOW / MEDIUM / HIGH)
- `plAvgDepth`: 평균 노출 광고수

#### 3.2.3 경쟁사 분석 (상위 40개 상품명)

```
GET /v1/search/shop.json?query=젤펜&display=40&sort=sim

→ 상위 40개 상품의 title 수집
→ 각 상품명에서 키워드 추출
→ 빈도수 분석 → 1페이지에서 많이 쓰이는 키워드 = 적합도 높은 키워드
```

**왜 중요한가:**
- 실제로 1페이지에 노출되는 상품들이 쓰는 키워드
- 네이버가 "이 키워드에 적합하다"고 판단한 상품들
- 우리도 비슷한 키워드 구성을 해야 적합도 높음

## 4. 경쟁강도 기반 단계별 확장 전략

### 4.1 핵심 원리

```
경쟁강도 높은 키워드 = 검색량 많음 + 경쟁 치열 + 인기도 필요
경쟁강도 낮은 키워드 = 검색량 적음 + 경쟁 적음 + 인기도 적어도 OK

→ 처음엔 경쟁강도 낮은 곳에서 1페이지 먹고 인기도 쌓기
→ 인기도 쌓이면 경쟁강도 높은 곳으로 확장
```

### 4.2 단계별 키워드 선정 기준

#### [1단계: 극초반] 경쟁강도 LOW 공략

```
조건:
  - compIdx = LOW
  - 월간 검색량 100~1,000
  - 현재 내 순위 확인 불가 or 500위+

목표:
  - 이 키워드에서 1페이지(40위) 진입
  - 소량이라도 클릭/전환 발생 → 인기도 축적 시작

예시 키워드:
  - "퀵드라이 젤펜 0.5mm"
  - "그립감 좋은 볼펜"
  - "손글씨 연습 펜"
```

#### [2단계: 성장기] 경쟁강도 MEDIUM 확장

```
조건:
  - compIdx = MEDIUM
  - 월간 검색량 1,000~10,000
  - 1단계 키워드에서 1페이지 유지 중

목표:
  - 중간 키워드에서도 1페이지 진입
  - 인기도 추가 축적

예시 키워드:
  - "선물용 볼펜"
  - "퀵드라이 젤펜"
  - "중성펜 추천"
```

#### [3단계: 안정기] 경쟁강도 HIGH 도전

```
조건:
  - compIdx = HIGH
  - 월간 검색량 10,000+
  - 2단계 키워드에서 1페이지 유지 중

목표:
  - 대형 키워드에서 1페이지 진입
  - 대량 유입 확보

예시 키워드:
  - "젤펜"
  - "볼펜"
```

### 4.3 확장 판단 기준

```
현재 단계에서 다음 단계로 확장 가능 조건:

1. 현재 단계 키워드 중 50% 이상 1페이지 유지
2. 1페이지 유지 기간 7일 이상
3. 해당 키워드에서 클릭/전환 발생 확인
```

## 5. 키워드 라이프사이클 관리

### 5.1 라이프사이클 단계

```
[발굴] → [테스트] → [유지] → [교체/확장]
                ↓
            [실패]
```

### 5.2 단계별 상세

#### 5.2.1 발굴 (Discovery)

```
소스:
  - 검색광고 연관 키워드 API
  - 상위 40개 경쟁사 상품명 분석
  - 내 상품명 토큰화

필터링:
  - 현재 인기도 단계에 맞는 경쟁강도
  - 이미 테스트한 키워드 제외
  - 현재 운영 중인 키워드 제외
```

#### 5.2.2 테스트 (Testing)

```
방법:
  - 상품명 또는 태그에 키워드 반영
  - 7~14일간 순위 추적

성공 기준:
  - 1페이지(40위) 진입

실패 기준:
  - 14일 후에도 100위 밖
  - 순위 지속 하락
```

#### 5.2.3 유지 (Maintenance)

```
조건:
  - 1페이지 진입 성공

모니터링:
  - 일간 순위 추적
  - 클릭/전환 발생 여부
  - 인기도 기여도 평가

위험 신호:
  - 순위 급락 (전일 대비 20위+ 하락)
  - 1페이지 이탈
```

#### 5.2.4 교체/확장 (Replace/Expand)

```
교체 조건:
  - 1페이지 이탈 후 7일간 복귀 실패
  - 인기도 기여 없음 (클릭/전환 0)

확장 조건:
  - 현재 키워드 1페이지 안정적 유지
  - 인기도 축적 확인
  → 다음 단계(더 큰 경쟁강도) 키워드로 확장
```

### 5.3 키워드 상태 관리

| 상태 | 설명 | 액션 |
|------|------|------|
| `candidate` | 발굴된 후보 | 우선순위 평가 |
| `testing` | 테스트 중 (14일) | 순위 추적 |
| `active` | 1페이지 유지 중 | 모니터링 |
| `warning` | 순위 하락 중 | 원인 분석 |
| `failed` | 테스트 실패 | 제거 또는 재시도 대기 |
| `retired` | 교체됨 | 히스토리 보관 |

## 6. 인기도 기여 키워드 판별

### 6.1 왜 중요한가

1페이지에 있는 키워드가 여러 개라도, **실제로 인기도에 기여하는 키워드**는 일부

```
예시:

| 키워드 | 순위 | 월간 검색량 | 실제 클릭 | 기여도 |
|--------|------|------------|----------|--------|
| "퀵드라이 젤펜 0.5" | 3위 | 100 | 5 | 낮음 |
| "선물용 볼펜" | 15위 | 2,000 | 150 | 높음 ⭐ |
| "젤펜" | 500위 | 50,000 | 0 | 없음 |
```

→ **"선물용 볼펜"**이 인기도에 가장 기여
→ 이 키워드는 죽어도 지켜야 함

### 6.2 기여도 판별 기준

```
인기도 기여도 = f(순위, 검색량, 실제 유입)

High 기여:
  - 1페이지 + 검색량 1,000+ + 클릭 발생

Medium 기여:
  - 1페이지 + 검색량 100~1,000 + 클릭 발생

Low 기여:
  - 1페이지 + 검색량 100 미만
  - 또는 클릭 거의 없음

No 기여:
  - 2페이지 이하
```

### 6.3 기여도별 관리 전략

| 기여도 | 전략 |
|--------|------|
| **High** | 절대 보호. 상품명에서 절대 제거 금지 |
| **Medium** | 유지. 더 높은 기여도 키워드로 대체 가능 |
| **Low** | 공간 확보 위해 제거 고려 |
| **No** | 제거 또는 다른 키워드로 교체 |

## 8. 프로세스 흐름

### 8.1 전체 흐름

```
[Phase 1: 키워드 발굴]
     │
     ├─ 검색광고 연관 키워드 조회
     ├─ 상위 40개 경쟁사 상품명 분석
     └─ 경쟁강도 + 검색량 기반 필터링
           │
           ▼
[Phase 2: 1페이지 가능성 판단]
     │
     ├─ 현재 인기도 단계 확인
     ├─ 경쟁강도 vs 인기도 비교
     └─ 1페이지 가능성 점수 산출
           │
           ▼
[Phase 3: 키워드 반영]
     │
     ├─ 상품명에 키워드 추가/수정
     └─ 태그에 키워드 추가
           │
           ▼
[Phase 4: 순위 추적]
     │
     ├─ 일간 순위 수집
     ├─ 1페이지 진입 여부 판정
     └─ 테스트 성공/실패 판정
           │
           ▼
[Phase 5: 인기도 기여 평가]
     │
     ├─ 1페이지 키워드 중 기여도 평가
     ├─ High 기여 키워드 보호
     └─ Low 기여 키워드 교체 검토
           │
           ▼
[Phase 6: 확장/순환]
     │
     ├─ 인기도 축적 확인
     ├─ 다음 단계 키워드 발굴
     └─ 반복
```

### 8.2 일일 자동화 실행

```
[매일 06:00 KST]

1. 순위 수집
   - 모든 추적 키워드의 현재 순위 조회
   - 전일 대비 변화량 계산
   - 1페이지 진입/이탈 감지

2. 상태 업데이트
   - testing → active (1페이지 진입 시)
   - active → warning (순위 급락 시)
   - testing → failed (14일 후 실패 시)

3. 키워드 발굴 (주 1회)
   - 연관 키워드 조회
   - 경쟁사 상품명 분석
   - 후보 키워드 추가

4. 리포트 발송
   - 1페이지 진입/이탈 알림
   - 기여도 변화 알림
   - 다음 확장 가능 여부 알림
```

## 9. 키워드 유형 분류와의 연계

> 참고: [keyword-analysis-logic.md](./keyword-analysis-logic.md)

키워드 유형 분류(조합형/일체형/순서고정)는 **상품명 작성 규칙**에 영향:

| 유형 | 상품명 작성 규칙 |
|------|-----------------|
| **조합형** | 순서 무관, 공통 요소 1회만 |
| **일체형** | 반드시 붙여서 기재 |
| **순서고정** | 원래 순서 유지 필수 |

**전략 연계:**
1. 키워드 발굴 시 유형 분류 실행
2. 상품명 최적화 시 유형별 규칙 적용
3. 조합형은 효율적 배치로 여러 키워드 노출 가능

## 10. 색깔 분류와의 연계

> 참고: [system-design-v2.md](./system-design-v2.md) Section 3

색깔 분류는 **키워드 배치 위치(상품명/태그)**에 영향:

| 색깔 | 의미 | 배치 |
|------|------|------|
| 🟡 상품명전용 | 상품명에 있어야 노출 | 상품명 필수 |
| 🟢 속성 | 속성으로도 노출 가능 | 인기도 높으면 상품명 |
| 🔵 태그 | 태그로 노출 가능 | 태그 배치 |
| ⚪ 카테고리 | 카테고리에서 노출 | 초반 사용 금지 |

**전략 연계:**
1. 발굴된 키워드의 색깔 분류 실행
2. 🟡 키워드 우선 상품명 배치
3. 현재 인기도 단계에 맞는 색깔 키워드만 사용
